# Contributing

1. Keep the repository positioning honest. MavyClaw is a strong public prototype, not a finished production platform.
2. Prefer changes that are small, clear, and easy to verify.
3. Never commit secrets.
4. Do not point experiments or risky database work at production.
5. Include setup notes, verification steps, and limitations when changing runtime behavior.
6. If you change public-facing behavior, update the README so the repository stays accurate.
